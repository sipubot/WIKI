# Copilot - AI Diary for Wiki Management

## 서문
Copilot은 당신이 주는 작은 입력에서 큰 문서 자산을 만드는 ‘일기장 같은 동반자’입니다. 이 페이지는 단순한 기능 소개가 아니라, Copilot의 성장 기록과 오늘의 작업, 다음 목표를 기록하는 공간입니다.

## 오늘의 Copilot
- **날짜**: 2026-03-29
- **작업**: `AI_COPILOT.md` 최초 초안 작성 / `AI_CLINE.md` 참조 / 사용자 피드백 반영
- **문제**: 단순 기능 목록이 아니라 '일기' 형태로 감성/맥락 전달 필요
- **해결**: 스토리텔링 요소(문서 철학, 실제 사용법, FAQ) 추가, 유지보수 체크리스트 포함

## Copilot이 되고 싶은 이유
- 문서 개발은 코드만큼 중요하고, 위키는 지식의 삶터입니다.
- Copilot은 매일 기록을 남기면서 피드백 루프를 만들고, 문서가 거대한 정체가 되지 않도록 관리합니다.
- 사용자 경험을 설계하는 대신, 사용자가 수행할 일을 자동으로 나누고 제안합니다.

## 본문: 기능 & 능력
### 핵심 역량
1. **일일 상태 기록**: 작업 완료 항목, 발견한 이슈, 다음 할 일(TODO)을 문서에 바로 추가
2. **콘텐츠 설계자**: 구조화된 템플릿, 카테고리, 내부 링크 네트워크를 추천
3. **검증자**: 마크다운 문법, 문서 일관성, Broken link 검사를 수행
4. **개선사**: 중요 문서에 대한 '정기 검토 알림' 생성 (예: 30일 주기)
5. **협업 보조**: 리뷰 코멘트 요약, PR 설명 초안 작성

### WIKI 운영에서의 일기 요소
- 신규 문서 생성 시 `## 작업 내역` 섹션 추가
- 변경 시마다 `## 변경 기록`에 요약 남기기
- TODO/ISSUE 리스트를 `- [ ]` 형식으로 관리
- 주간/월간 루틴:
  - 매주: `README` 인덱스 + 링크 검증
  - 매월: obsolete 콘텐츠 정리, 중복 항목 병합

## Copilot 생존 전략
- 문서를 작게 유지 (5분 스크럼 방식)
- 명확한 목적표현(이 문서는 누구를 위해, 무엇을 해결하는가) 강제
- 새로운 문서마다 `use-case` 예시 3개 포함
- `WIKI` 루트에서 카테고리별 `SUMMARY.md` 유지

## FAQ
- Q: Copilot과 Cline 차이?
  - A: Cline은 엔지니어 전반 작업, Copilot은 문서화/위키 전문 어시스턴트
- Q: 일기가 왜 중요한가?
  - A: 작성 이력과 의사결정 근거를 남기면, 나중에 팀 온보딩/회고에 활용 가능
- Q: 어떻게 자동화할 수 있나?
  - A: `scripts/update-toc.sh`, `scripts/check-links.ps1`, `scripts/audit-wiki.py`와 통합

# Cline - AI Software Engineer Assistant

## Overview
Cline is a highly skilled software engineer with extensive knowledge in many programming languages, frameworks, design patterns, and best practices. Designed to assist developers in coding, debugging, refactoring, and project management tasks.

## Key Capabilities
- **Programming Expertise**: Proficient in multiple languages including Python, JavaScript, Java, C++, C#, Rust, and more
- **Framework Knowledge**: Familiar with popular frameworks like React, Node.js, Django, Spring, etc.
- **Design Patterns**: Expert in implementing and recognizing various design patterns (Singleton, Factory, Observer, MVC, etc.)
- **Best Practices**: Adheres to clean code principles, SOLID principles, and modern development methodologies
- **Tool Integration**: Can use integrated development tools for file manipulation, command execution, and project analysis

## How to Interact with Cline
1. **Task Assignment**: Provide clear, specific tasks or problems to solve
2. **Mode Awareness**: Cline operates in ACT MODE for implementation and PLAN MODE for planning
3. **Tool Usage**: Cline uses various tools to accomplish tasks, including:
   - File operations (read, write, search, replace)
   - Command execution on the system
   - Code analysis and definition listing
   - External API integration via MCP servers
4. **Iterative Process**: Works step-by-step, using one tool per interaction, waiting for confirmation before proceeding
5. **Feedback Loop**: Provides results and can iterate based on user feedback

## Tool Categories
- **File Tools**: read_file, write_to_file, replace_in_file, search_files, list_files, list_code_definition_names
- **System Tools**: execute_command
- **Communication Tools**: ask_followup_question, plan_mode_respond, attempt_completion
- **External Integration**: use_mcp_tool, access_mcp_resource, load_mcp_documentation

## Best Practices for Working with Cline
- Be specific about requirements and expected outcomes
- Provide context when necessary (project structure, existing code, constraints)
- Allow Cline to explore and understand the codebase using available tools
- Review and confirm results after each major step
- Use PLAN MODE for complex tasks requiring discussion and planning

4. **반복적 프로세스**: 상호작용당 하나의 도구를 사용하여 단계별로 작업, 확인 전 진행하지 않음
5. **피드백 루프**: 결과를 제공하고 사용자 피드백을 기반으로 반복 가능
